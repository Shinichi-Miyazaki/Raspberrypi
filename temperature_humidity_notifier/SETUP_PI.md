# Raspberry Pi 初期構築手順書（温湿度監視システム用）

**この手順書の役割**: 何も入っていないRaspberry Piを、
**SSHで接続できてセンサーが配線された状態**にするまでを扱います。

**この手順書が終わったら**: 続けて [OPERATIONS.md](OPERATIONS.md) のフェーズ1に進んでください。
Slack・NAS・プログラムの設定はすべてそちらに書いてあります。

**対象読者**: パソコンの基本操作はできるが、Raspberry Pi や Linux は初めての方。

## この手順書が必要になる場面

- Piを新しく増やす・壊れたPiを交換する
- microSDカードが壊れてOSを入れ直す
- システムを一から作り直す

**すでにPiが動いていてSSH接続できる場合、この手順書は不要です。**
[OPERATIONS.md](OPERATIONS.md) から始めてください。

### 状態確認（この手順書が必要か調べる）

自分のパソコンのPowerShell（Windows）またはターミナル（Mac）で:

```
ssh 【Piのユーザー名】@【そのPiのIPアドレス】
```

接続できて `【Piのユーザー名】@【ホスト名】:~ $` のような表示が出る
→ **このPiは構築済み**。この手順書は飛ばして OPERATIONS.md へ

---

## 設定値メモ（記入・更新すること）

OPERATIONS.md の「設定値メモ」と**同じ値を使います**。両方を揃えてください。

| 項目 | 値（記入欄） | 例 |
|---|---|---|
| 【Piのユーザー名】 | ___________ | th-meter |
| 【ホスト名】（Piごとに変える） | ___________ | pi-zero-01 〜 04 |
| 【Wi-FiのSSID】 | ___________ | GL-SFT1200-xxx |
| 【Piのパスワード】 | 管理者が保管 | （紙やメールに残さない） |

**【ホスト名】はPiごとに違う名前**にしてください（`pi-zero-01`, `pi-zero-02` …）。
同じ名前のPiが2台あると、ネットワーク上で区別できなくなります。

---

## 用意するもの（Pi 1台あたり）

| 品目 | 補足 |
|---|---|
| Raspberry Pi 本体 | Pi Zero WH / 3B+ / 4 のいずれでも動作する |
| microSDカード（16GB以上） | 書き込み用のカードリーダーも必要 |
| 電源アダプタ | Piの機種に合ったもの |
| DHT22 温湿度センサー | モジュール品（基板付き）が配線しやすい |
| ジャンパーワイヤー（メス-メス または オス-メス）3本 | センサーの端子形状に合わせる |
| 4.7kΩ または 10kΩ 抵抗 | **モジュール品なら不要**（内蔵されているため） |

Wi-Fi環境: 各Piは**NASと同じネットワーク**につながる必要があります。
研究室のWi-Fiが届かない部屋では中継機（GL.iNet等）を使います（後述のPitfalls参照）。

---

# 1. microSDカードを作る（自分のパソコンで作業）

**所要時間**: 約20分（書き込み待ち時間を含む）

## 1-1. Raspberry Pi Imager をインストール

1. https://www.raspberrypi.com/software/ を開く
2. 自分のパソコンのOS用（Windows / macOS）のインストーラをダウンロードして実行
3. インストール後、Raspberry Pi Imager を起動する

## 1-2. 書き込む内容を選ぶ

1. microSDカードをパソコンに挿す
2. Imagerの画面で3つを選ぶ:
   - **デバイス**: 使うPiの機種（例: Raspberry Pi Zero）
   - **OS**: `Raspberry Pi OS Lite (32-bit)` を選ぶ。
     **Lite（画面なし版）で正しい**です。このシステムは画面を使わないため、
     デスクトップ版を選ぶと余計なソフトが入ってPiが遅くなります
   - **ストレージ**: 挿したmicroSDカード。
     **⚠ 間違えて外付けHDDやUSBメモリを選ぶと中身が全部消えます。**
     容量表示を見て、確実にmicroSDカードであることを確認してください

## 1-3. 詳細設定【最重要】

「次へ」を押すと「OSカスタマイズ設定を適用しますか？」と聞かれるので
**「設定を編集する」** を選びます（ここを飛ばすとSSH接続できず、
最初からやり直しになります）。

**一般** タブ:

| 項目 | 設定する内容 |
|---|---|
| ホスト名 | 【ホスト名】（例: `pi-zero-01`）。**Piごとに変える** |
| ユーザー名 | 【Piのユーザー名】（例: `th-meter` = Temperature Humidity Meter の略） |
| パスワード | 管理者から指定されたもの |
| Wi-Fi の SSID | 【Wi-FiのSSID】 |
| Wi-Fi のパスワード | そのWi-Fiのパスワード |
| Wi-Fi を使う国 | `JP` |
| ロケール設定 | タイムゾーン `Asia/Tokyo` |

**サービス** タブ:

- **SSHを有効化する** に必ずチェック
- 「パスワード認証を使う」を選ぶ

**⚠ Wi-Fiの入力ミスに注意**: SSIDとパスワードは、大文字・小文字、
全角・半角（特にハイフンとアンダースコア）を1文字でも間違えると
**Piがネットワークに現れず、原因の調べようがありません**。
あとから直すこともできず、microSDの書き込みからやり直しになります。
可能ならコピー＆ペーストで入力してください。

## 1-4. 書き込む

1. 「保存」→「はい」→「はい」（データが消える確認）で書き込みが始まる
2. 完了メッセージが出るまで待つ（10分程度）
3. microSDカードをパソコンから取り外す

---

# 2. Piを起動する

1. Piの電源を**抜いた状態**で、microSDカードをPiに挿す
2. センサーの配線をする（次の「3. センサーの配線」を先に読む）
3. 電源を接続する
4. **3分間、何もせずに待つ**。初回起動はファイルシステムの拡張などで
   時間がかかります。この間にLEDが点滅したり消えたりするのは正常です

---

# 3. センサーの配線

**⚠ 配線は必ずPiの電源を抜いた状態で行ってください。**
通電中に配線するとPiやセンサーが壊れることがあります。

DHT22センサーとPiをジャンパーワイヤーでつなぎます。

| センサー側 | Pi側 | 物理ピン番号 |
|---|---|---|
| **VCC（+ / VDD）** | 5V電源 | Pin 2 または Pin 4 |
| **DATA（OUT / S）** | GPIO 4 | **Pin 7** |
| **GND（- / GND）** | グランド | Pin 6, 9, 14, 20, 25, 30, 34, 39 のどれか |

補足:

- **物理ピン番号とGPIO番号は別物です**。`GPIO 4` は基板の端から数えて
  **7番目のピン**（Pin 7）を指します。「ピン4番」ではありません
- ピン番号の数え方は `pinout` というコマンドでPi上に図が表示されます
  （SSH接続後に実行できます）
- **モジュール品でないDHT22**（黒い部品単体）を使う場合のみ、
  DATAピンと5Vの間に 4.7kΩ の抵抗（プルアップ抵抗）が必要です。
  基板付きのモジュール品には内蔵されているので不要です
- **プログラムはGPIO 4に固定**されています。他のピンにつなぐと動きません

配線したら、もう一度目で確認してから電源を入れてください。

# 4. SSHで接続する

自分のパソコンから、ネットワーク越しにPiを操作できるようにします。

## 4-1. 接続する

Windowsなら「PowerShell」、Macなら「ターミナル」を起動して:

```
ssh 【Piのユーザー名】@【ホスト名】.local
```

例: `ssh th-meter@pi-zero-01.local`

1. 初回は `Are you sure you want to continue connecting?` と聞かれるので
   `yes` と入力してEnter
2. パスワードを聞かれたら入力してEnter。
   **パスワードは打っても画面に何も表示されません（●も出ない）。
   故障ではないので、そのまま打ち切ってEnterを押してください**
3. 次のような表示に変われば成功です:
   ```
   th-meter@pi-zero-01:~ $
   ```

つながらない場合は、この手順書の末尾「よくある失敗と対策」を見てください。

**Tera Term でも接続できます**（Windowsで使い慣れている場合）。
ホストに `【ホスト名】.local`、サービスに SSH を選びます。

## 4-2. IPアドレスを調べて固定する

`.local` という名前での接続は、ネットワークによっては使えないことがあります。
また、OPERATIONS.md ではIPアドレスを使うので、ここで調べておきます。

Piに接続した状態で:

```
$ hostname -I
```

`192.168.1.21` のような数字が表示されます。これがこのPiのIPアドレスです。
**OPERATIONS.md の「設定値メモ」に記入してください。**

**重要**: このIPアドレスは、放っておくとある日勝手に変わります。
変わるとNASのファイアウォールに弾かれてシステムが止まるため、
ルーターの管理画面で**固定（DHCP固定割当）**にしてください
（設定方法はルーターの機種によります。「【ルーター機種名】 DHCP 固定」で検索）。

## 4-3. 接続を切る

```
$ exit
```

# 5. 必要なソフトを入れる

Piに接続した状態で、上から順に実行します（コピー＆ペーストでOK）。

```
$ sudo apt-get update
$ sudo apt-get install -y python3-pip cifs-utils git
$ pip install adafruit-circuitpython-dht --break-system-packages
```

各コマンドの意味:

- `apt-get update`: インストールできるソフトの一覧を最新にする
- `cifs-utils`: NASのフォルダをPiから見えるようにするためのソフト
- `git`: リポジトリからプログラムを取得するためのソフト
- `adafruit-circuitpython-dht`: DHT22センサーを読むためのPythonライブラリ
- `--break-system-packages`: 最近のRaspberry Pi OSでpipを使うために必要な指定

**センサーPiにはこれだけで十分です。**
Slack関連やグラフ描画のライブラリ（`slack_sdk`, `matplotlib`）は**不要**です。
通知とグラフはコレクター役のPi1台だけが担当する設計のためです
（コレクター役に追加で入れるものは OPERATIONS.md のフェーズ4に書いてあります）。

## 動作確認（センサーが読めるか）

```
$ python3 -c "import adafruit_dht; import board; print(adafruit_dht.DHT22(board.D4).temperature)"
```

`23.4` のような数字が表示されれば、配線もライブラリも正常です。

- `RuntimeError` が出た → DHT22ではよくある一時的な読み取り失敗です。
  もう一度実行してみて、数字が出れば問題ありません
- 何度やってもエラー → 配線を確認してください（末尾「失敗3」参照）

# よくある失敗と対策

### 失敗1: `pi-zero-01.local` につながらない

**原因のほとんどはWi-Fi設定の入力ミス**です（SSIDかパスワードの1文字違い）。
Piはネットワークに現れず、外から原因を調べる方法がありません。

対策は、順に試します:

1. 電源を入れてから3分以上待ったか確認する（起動に時間がかかる）
2. 自分のパソコンが**Piと同じWi-Fi**につながっているか確認する
   （別のネットワークからは `.local` で見つけられません）
3. ルーターの管理画面で「接続中の機器一覧」を見る。
   【ホスト名】が載っていれば、そのIPアドレスに直接接続してみる:
   ```
   ssh 【Piのユーザー名】@192.168.1.21
   ```
4. 一覧にも載っていない → Wi-Fi設定のミスが濃厚。
   **microSDの書き込み（1章）からやり直す**のが最も速い解決です
   （書き込み済みのカードの設定だけを直すことはできません）

### 失敗2: `.local` で接続できないがIPでは接続できる

mDNS（`.local` の名前解決）がネットワークで使えない環境です。
**IPアドレスで接続すれば問題ありません**。4-2 で固定したIPを使ってください。

### 失敗3: センサーの値が読めない・エラーが続く

1. Piの電源を切る
2. 配線を確認する。特に **DATA が Pin 7（GPIO 4）** に挿さっているか。
   物理ピン4番に挿していることがよくある間違いです
3. ジャンパーワイヤーが奥まで挿さっているか確認する（緩みが多い）
4. 電源を入れ直して、動作確認コマンドをもう一度実行する
5. それでもダメなら、センサーを別のものに交換して試す
   （DHT22は個体差があり、初期不良もあります）

`unable to set line to input` というエラーの場合は配線ではなく、
GPIOが前のプログラムに掴まれたままです。`gpio_reset.py` で復旧できます
（OPERATIONS.md のフェーズ7参照）。

### 失敗4: 部屋にWi-Fiが届かない（中継機を使う場合）

GL.iNet等の小型ルーターを中継機として使う場合の注意:

- **必ず「アクセスポイントモード」または「Extenderモード」**にする。
  ルーターモードのままだと二重ルーターになり、
  **NASと別のネットワークに入ってしまってデータが送れません**
- アクセスポイントモードに切り替えると、**中継機の管理画面（192.168.8.1）に
  入れなくなります。これは仕様です**。設定を変えたい場合は、
  本体のリセットボタンを10秒以上長押しして工場出荷状態に戻します
- 中継機は居室と**違うSSID**（`GL-SFT1200-xxx` など）を出します。
  それで問題ありません。同じにしたい場合は、
  モード切替の**前**に管理画面でSSIDを変更しておきます

### 失敗5: microSDカードが壊れた（Piが起動しなくなった）

microSDカードは書き込みを繰り返すと数年で寿命が来ます。
この手順書の1章からやり直せば復旧できます。

**データの心配は不要です**: 測定データはNAS上のデータベースに入っており、
Piのカードには送信済みの控えしか入っていません。

# 困ったときの調べ方

### 検索キーワード

- `Raspberry Pi Imager SSH 有効化`
- `Raspberry Pi Zero Headless Setup`
- `raspberry pi .local 接続できない`
- `GL-SFT1200 アクセスポイントモード 設定`

### AIへの質問例

> Raspberry Pi Zero W をヘッドレス（画面なし）でセットアップしました。
> OSはRaspberry Pi OS Liteで、Imagerの設定でSSHとWi-Fiを有効にしたはずです。
> PCと同じWi-Fiにつないでいますが、`ssh th-meter@pi-zero-01.local` が
> タイムアウトします。確認すべきポイントを初心者向けに教えてください。
